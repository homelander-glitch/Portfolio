About Me

Who I Am

I'm Aakash — CSE third year, but I've been building things long enough to have strong opinions about it. Chess engines, mobile diagnostic tools, agentic AI security auditors. I work across frontend, AI pipelines, and system tooling. The logic always comes first. The rest follows.

What I Build

Tools that talk to real hardware and real systems — speed testers that measure actual network timing, audio tools that use raw oscillators, agents that read and rewrite code. If it doesn't do something, I'm not interested in shipping it.

How I Think
  • Logic working first. No UI until the core behaviour is solid.
  • Test on mobile early — that's where things break in interesting ways.
  • Web APIs directly — I'd rather understand MediaDevices than wrap a library I don't trust.
  • Glassmorphism is earned, not default.
Current Focus

Agentic AI systems — agents that can read a codebase, reason about it, and act on it safely inside a sandbox. Pairing that with the systems fundamentals I'm getting in class: OS, networks, databases.

Goals

Get good enough at the fundamentals that frameworks stop being magic. Ship projects that solve a real, specific problem rather than demo well and do nothing. Keep a GitHub history that actually reflects the work.

Favorite Technologies

Vanilla JavaScript and the browser's own APIs — Fetch, Web Audio, MediaDevices. On the AI side, LangChain and Docker for building agents that can safely execute real actions.

Learning Right Now

Multi-agent architectures — coordinating several specialized agents instead of one generalist. Also deepening OS and networks coursework so my tools aren't just calling APIs, they understand what's underneath them.
